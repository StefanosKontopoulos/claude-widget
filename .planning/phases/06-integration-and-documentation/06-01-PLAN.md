The current code only calls `ClaudeUsageWidget().updateAll(applicationContext)` inside the `isSuccess` branch. When fetchAndStore returns failure due to 401/403 (credentials already cleared by UsageRepository), the widget is never told to re-render, so it continues showing stale data until the next periodic update.

Change the failure branch to:
1. Check if the failure is an auth expiry (the exception message contains "Auth expired")
2. If auth expired: call `ClaudeUsageWidget().updateAll(applicationContext)` BEFORE returning `Result.retry()` so the widget immediately re-renders and reads the now-null credentials, showing "Sign in to Claude app"
3. For non-auth failures (network timeout, etc.): keep existing behavior (just `Result.retry()` without widget update -- cached data is still valid)

The resulting doWork() should look like:
```kotlin
override suspend fun doWork(): Result {
    Log.d(TAG, "Starting usage fetch (attempt $runAttemptCount)")
    val result = UsageRepository.fetchAndStore(applicationContext)
    return if (result.isSuccess) {
        Log.i(TAG, "Usage fetch succeeded, updating widget")
        ClaudeUsageWidget().updateAll(applicationContext)
        Result.success()
    } else {
        val error = result.exceptionOrNull()
        Log.w(TAG, "Usage fetch failed, scheduling retry", error)
        // On auth expiry, credentials are already cleared by UsageRepository.
        // Update widget immediately so it shows "Sign in" state.
        if (error?.message?.contains("Auth expired") == true) {
            Log.i(TAG, "Auth expired, updating widget to sign-in state")
            ClaudeUsageWidget().updateAll(applicationContext)
        }
        Result.retry()
    }
}
```

Current bug: `let hasCreds = CredentialStore.loadSessionCookie() != nil` is evaluated BEFORE `try? await UsageRepository.fetchAndStore()`. If the fetch receives a 401/403, UsageRepository clears credentials, but `hasCreds` is still `true` from the pre-fetch check. The timeline entry is created with `hasCredentials: true` even though credentials are gone. The widget won't show "Sign in" until 15 minutes later.

Fix: Move the credential check to AFTER the fetch attempt. The revised getTimeline should:

```swift
func getTimeline(in context: Context, completion: @escaping (Timeline<SimpleEntry>) -> Void) {
    Task {
        // Attempt fetch if credentials exist (fetch may clear them on 401/403)
        if CredentialStore.loadSessionCookie() != nil {
            try? await UsageRepository.fetchAndStore()
        }
        // Re-check credentials AFTER fetch -- they may have been cleared on auth failure
        let hasCreds = CredentialStore.loadSessionCookie() != nil
        let data = UsageRepository.getCached()
        let entry = SimpleEntry(date: .now, usageData: data, hasCredentials: hasCreds)
        let nextUpdate = Calendar.current.date(byAdding: .minute, value: 15, to: .now)!
        let timeline = Timeline(entries: [entry], policy: .after(nextUpdate))
        completion(timeline)
    }
}
```

Key change: The `hasCreds` variable is now assigned AFTER fetchAndStore completes, so if the fetch cleared credentials due to 401/403, `hasCreds` will be `false` and the entry will show "Sign in to Claude app".

<objective>
Create the Android background refresh pipeline: a CoroutineWorker that fetches usage data every 15 minutes via WorkManager, updates the Glance widget on success, and retries on failure.

Purpose: BG-01 and BG-02 require a periodic background fetch that triggers widget updates without user interaction. WorkManager is the only reliable Android mechanism for guaranteed periodic background work.

Output: UsageFetchWorker, minimal Glance widget stub (for updateAll calls), widget receiver, widget metadata XML, WorkManager scheduling in MainActivity.
</objective>

    2. **Modify `MainActivity.kt`**:
       - Add imports:
         ```kotlin
         import androidx.work.ExistingPeriodicWorkPolicy
         import androidx.work.PeriodicWorkRequestBuilder
         import androidx.work.WorkManager
         import com.claudewidget.worker.UsageFetchWorker
         import java.util.concurrent.TimeUnit
         ```
       - Add a private method `scheduleUsageFetch()`:
         ```kotlin
         private fun scheduleUsageFetch() {
             val request = PeriodicWorkRequestBuilder<UsageFetchWorker>(
                 15, TimeUnit.MINUTES
             ).build()
             WorkManager.getInstance(this).enqueueUniquePeriodicWork(
                 UsageFetchWorker.WORK_NAME,
                 ExistingPeriodicWorkPolicy.KEEP,
                 request
             )
             Log.i(TAG, "WorkManager: scheduled periodic usage fetch")
         }
         ```
       - Call `scheduleUsageFetch()` at the end of `onCreate()` (after `setContentView`).
       - Use `ExistingPeriodicWorkPolicy.KEEP` so that relaunching the app does not reset the schedule. This prevents duplicate workers.
       - Do NOT wrap in any credential check -- the worker itself will gracefully fail (Result.retry) if credentials are missing, and scheduling should be in place before login completes.
